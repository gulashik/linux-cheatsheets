# find — шпаргалка

## Синтаксис

```
find [ГДЕ] [ПАРАМЕТРЫ] [ТЕСТ/ФИЛЬТР] [ДЕЙСТВИЕ]
```

---

## Где искать

| Путь | Значение |
|------|----------|
| `.` | текущая директория |
| `~` | домашняя директория |
| `/` | корень файловой системы |
| `/var/log` | конкретный путь |
| `/home /tmp` | несколько путей сразу |

---

## Фильтры по имени

| Флаг | Описание | Пример |
|------|----------|--------|
| `-name "pattern"` | имя файла (регистр важен) | `-name "*.log"` |
| `-iname "pattern"` | имя без учёта регистра | `-iname "*.TXT"` |
| `-path "*/dir/*"` | фильтр по полному пути | `-path "*/node_modules/*"` |
| `-not -path "…"` | исключить путь | `-not -path "*/.git/*"` |

---

## Фильтры по типу `-type`

| Флаг | Тип |
|------|-----|
| `-type f` | обычный файл |
| `-type d` | директория |
| `-type l` | символическая ссылка |
| `-type p` | именованный канал (pipe) |
| `-type s` | сокет |
| `-type b` | блочное устройство |
| `-type c` | символьное устройство |

---

## Фильтры по времени

> `-` = меньше чем, `+` = больше чем, без знака = ровно

| Флаг | Описание | Пример |
|------|----------|--------|
| `-mtime N` | изменён N дней назад | `-mtime -7` (за 7 дней) |
| `-atime N` | доступ N дней назад | `-atime +30` |
| `-ctime N` | изменён inode N дней назад | `-ctime -1` |
| `-mmin N` | изменён N минут назад | `-mmin -60` |
| `-amin N` | доступ N минут назад | `-amin +120` |
| `-newer file` | новее указанного файла | `-newer /etc/passwd` |

---

## Фильтры по размеру `-size`

> Единицы: `c` (байты), `k` (КБ), `M` (МБ), `G` (ГБ)

| Пример | Описание |
|--------|----------|
| `-size +100M` | больше 100 МБ |
| `-size -1k` | меньше 1 КБ |
| `-size +1G` | больше 1 ГБ |
| `-empty` | пустые файлы/директории |

---

## Фильтры по правам и владельцу

| Флаг | Описание | Пример |
|------|----------|--------|
| `-perm 755` | точные права | `-perm 644` |
| `-perm -4000` | установлен SUID бит | `-perm -4000` |
| `-perm /u+x` | исполняемые для владельца | `-perm /a+x` |
| `-user name` | владелец файла | `-user root` |
| `-group name` | группа файла | `-group www-data` |
| `-nouser` | файлы без владельца | — |
| `-nogroup` | файлы без группы | — |

---

## Глубина поиска

| Флаг | Описание |
|------|----------|
| `-maxdepth N` | не глубже N уровней |
| `-mindepth N` | не мельче N уровней |

```bash
find . -maxdepth 1 -type f        # только в текущей директории
find . -mindepth 2 -maxdepth 3    # 2–3 уровня вложенности
```

---

## Действия

| Флаг | Описание |
|------|----------|
| `-print` | вывести путь (по умолчанию) |
| `-print0` | вывести с \0 (для xargs -0) |
| `-ls` | вывести как ls -dils |
| `-delete` | удалить найденные файлы ⚠️ |
| `-exec CMD {} \;` | выполнить команду для каждого |
| `-exec CMD {} +` | выполнить одной командой (быстрее) |
| `-ok CMD {} \;` | как -exec, но с подтверждением |

```bash
# {} — подстановка найденного файла
find . -name "*.tmp" -exec rm -v {} \;
find . -name "*.log" -exec gzip {} +
```

---

## Логические операторы

| Оператор | Описание |
|----------|----------|
| `-and` / (умолч.) | логическое И |
| `-or` / `-o` | логическое ИЛИ |
| `-not` / `!` | отрицание |
| `\( … \)` | группировка |

```bash
find . \( -name "*.jpg" -o -name "*.png" \) -size +1M
find . -not -name "*.log"
find . -type f -not -path "*/.git/*"
```

---

## Перенаправление ошибок

```bash
find / -name "secret" 2>/dev/null          # скрыть ошибки доступа
find / -name "secret" 2>&1 | grep -v "Permission denied"
```

---

## Комбинация с xargs

```bash
# медленно (1 процесс на файл)
find . -name "*.py" -exec grep -l "TODO" {} \;

# быстро (пакетная обработка)
find . -name "*.py" | xargs grep -l "TODO"

# безопасно для имён с пробелами
find . -name "*.py" -print0 | xargs -0 grep -l "TODO"
```

## Практические примеры

```bash
# Найти и удалить логи старше 30 дней
find /var/log -name "*.log" -mtime +30 -delete

# Найти SUID файлы (аудит безопасности)
find / -perm -4000 -type f 2>/dev/null

# Найти большие файлы
find . -type f -size +100M -exec ls -lh {} \;

# Удалить пустые директории
find . -type d -empty -delete

# Найти файлы изменённые за последний час
find . -type f -mmin -60

# Исключить node_modules и .git
find . -not \( -path "*/node_modules/*" -o -path "*/.git/*" \) -name "*.js"

# Найти файлы без владельца
find / -nouser -o -nogroup 2>/dev/null

# Скопировать найденные файлы в директорию
find . -name "*.conf" -exec cp {} /backup/ \;

# Подсчитать количество файлов по типу
find . -type f -name "*.py" | wc -l

# Найти дубликаты по имени
find . -type f -name "*.jpg" | sort | uniq -d
```

## Различия: Linux vs macOS

| Особенность | GNU find (Linux) | BSD find (macOS) |
|-------------|-----------------|------------------|
| `-perm /u+x` | ✅ | ❌ используй `-perm +u+x` или `-perm +111` |
| `-printf` | ✅ | ❌ не поддерживается |
| `-maxdepth` | ✅ до фильтров | ✅ |
| Установить GNU find | — | `brew install findutils` → `gfind` |

> На macOS для полной совместимости: `brew install findutils` и используй `gfind`.

---

## Best practices

- Всегда проверяй командой без `-delete` перед удалением
- Используй `-print0 | xargs -0` для файлов с пробелами
- `-exec CMD {} +` быстрее чем `-exec CMD {} \;`
- Ограничивай глубину `-maxdepth` для ускорения поиска
- `2>/dev/null` убирает мусорные ошибки Permission denied
